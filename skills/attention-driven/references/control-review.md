# Control Review

Use this review when importing an external theory, adding a major method, or
changing the attention-driven skill itself. It is not a daily workflow. It is a
guard against copying forms without absorbing the operating principle.

Run three independent roles:

- Engineering-control expert: protects the borrowed principle.
- Skill author: protects the 30/70 law, stable entrypoint, and low burden.
- Target user: protects daily usability across code, design, writing, research,
  and project steering.

Each role answers:

1. What has already been absorbed correctly?
2. What is still missing?
3. What has formalism or burden risk?
4. What concrete changes would improve the skill?

Then synthesize:

```markdown
Control review synthesis:
- Shared conclusion:
- Disagreements:
- Adopt now:
- Reject or defer:
- Files to change:
```

Adopt only changes that improve local steering: clearer target, sharper system
shape, better observation, smaller correction, better disturbance handling, or
cleaner continuity for the next agent.

Do not preserve the full discussion unless it becomes load-bearing. A short
synthesis in a blueprint or record is usually enough.
