# attention-driven:reframe

Reframe layer: shape work before the category has stable precedent.
Reframe owns lens replacement: the old category vocabulary cannot guide the
work, so the project needs new primitives before design can be stable.

Use when the user asks for AI-native, agent-first, post-mobile, "rebuild X from
scratch under Y", "what should X look like now", or when a proposed design is
just old vocabulary with a new surface.

Read `references/reframe.md`, `references/traps.md`, and
`references/templates.md` when creating or closing concept artifacts.

## init

Set up `concepts/` only when a project will carry multi-session conceptual
exploration. Do not force concepts for one-off brainstorming.

If design docs already exist, add only a light crosswalk: concepts are upstream
of design and graduate into `design/DESIGN.md` only when the skeleton stabilizes.

## default

For a new reframe:

1. Name the target and the old paradigm being escaped.
2. Extract 3-5 abstract functions the system must perform.
3. Draw a new skeleton from the new paradigm's primitives.
4. Stress-test by removing traditional crutches.
5. Add familiar flesh only as projection from the skeleton.
6. Run a comprehension test with the lightest credible artifact.

Keep the concept document as the working surface if the exploration spans
sessions.

## explain

When asked to explain a reframe, tailor the explanation to the audience and
focus. Do not restate every phase; translate the skeleton into what that
audience needs to understand or decide.

## close

Close a concept when it is adopted into design, abandoned, or superseded.

- Adopted: fold load-bearing skeleton into `design/DESIGN.md` or a design
  decision, then archive/mark the concept closed.
- Abandoned: record why it did not pay off.
- Superseded: point to the successor concept.

Closure should preserve the lesson, not polish every historical note.
